# EXPERIMENT_REGISTER — реестр экспериментов TwoComms

> Статус документа: рабочий реестр. Ни один эксперимент не является «решённым» до получения собственных данных TwoComms.
> Связанные документы: `MASTER_PLAN_V2.md` (разделы 5, 7, 9, 10), `ANALYTICS_AND_TRACKING_BLUEPRINT.md`, `DATA_GAPS_FIRST_14_DAYS.md`.

## Как пользоваться реестром

Каждый эксперимент имеет:

- **ID** — уникальный код (EXP-XX).
- **Гипотеза** — формулировка «если X, то Y, потому что Z». Гипотеза — не факт.
- **Первичная метрика** — одна метрика, по которой принимается решение. Почти всегда это CM2 на уровне маршрута/продукта, а не CTR/CPM/CPL.
- **Guardrail-метрики** — метрики, которые не должны деградировать.
- **Предусловия** — что должно быть готово до запуска (зависимости, а не даты).
- **Дизайн** — как проводится, сколько данных нужно.
- **Стоп-условие** — при чём эксперимент останавливается досрочно.
- **Критерий решения** — что считается «подтверждено / опровергнуто / нужно больше данных».
- **Статус** — planned / running / paused / concluded.

Правила:

1. Не более 1–2 активных платных экспериментов одновременно (бюджет и внимание владельца ограничены).
2. Эксперимент без заранее записанного критерия решения не запускается.
3. Результат записывается в реестр даже если он «неудобный».
4. Малые выборки: при < 30–50 целевых событий на вариант вывод записывается как «недостаточно данных», а не как «вариант A лучше».

## Блок A — Meta Ads и маршруты

### EXP-01 — Первый узкий запуск Meta: Ready B2C → Direct

- **Гипотеза:** реклама готовых футболок с объективом Messaging Conversations даёт переписки, которые конвертируются в оплаченные и выкупленные заказы с положительным CM2 на чат.
- **Первичная метрика:** CM2 на доставленный заказ из чата; вспомогательно — стоимость квалифицированной переписки.
- **Guardrails:** доля мусорных чатов; время владельца на чат; RTS.
- **Предусловия:** Phase 0 из MASTER_PLAN_V2 §7 (пиксель/CAPI, UTM, маркировка переписок, CRM-теги, таблица учёта чатов), заполненная себестоимость SKU (CM0/CM1), cash buffer.
- **Дизайн:** 1 кампания, 1–2 ad set, 3–4 креатива (creative screening, не A/B). Узкий набор реально готовых SKU. Бюджет не дробить.
- **Стоп-условие:** израсходован тестовый бюджет без единой квалифицированной переписки; либо чаты идут, но 0 заказов после N квалифицированных переписок (N зафиксировать до запуска).
- **Критерий решения:** экономика чата считается по факту: chats → qualified → order → paid → delivered → CM2. Решение о масштабировании только при положительном CM2 и свободной производственной мощности.
- **Статус:** planned.

### EXP-02 — Direct vs Site (маршрутный тест)

- **Гипотеза:** маршрут на сайт даёт худшую конверсию в первый заказ, но лучшую экономику времени владельца и масштабируемость; какой маршрут даёт больший CM2 на 1000 показов — неизвестно.
- **Первичная метрика:** CM2 на 1000 показов по каждому маршруту (включая стоимость времени владельца в Direct-маршруте).
- **Предусловия:** EXP-01 дал базовую линию Direct; сайт-backlog P0 из `SITE_IMPROVEMENT_BACKLOG.md` закрыт; e-commerce события работают (purchase, begin_checkout).
- **Дизайн:** одинаковые креативы и аудитория, разные destination. Последовательно или параллельно — в зависимости от бюджета; при малом бюджете лучше последовательные окна с одинаковой длительностью.
- **Стоп-условие:** технические проблемы трекинга на сайте (данные несравнимы) — сначала чинить, потом тестировать.
- **Критерий решения:** маршрут-победитель по CM2/1000 показов при сопоставимых объёмах; при недостатке данных — продлить, а не «объявить победителя».
- **Статус:** planned (после EXP-01).

### EXP-03 — Leads through Messaging / структурированная квалификация

- **Гипотеза:** переход от «просто переписок» к оптимизации на квалифицированный лид снижает долю мусорных чатов без критического роста цены.
- **Первичная метрика:** CM2 на квалифицированный лид.
- **Предусловия:** накоплен фактический процент квалификации из EXP-01; маркировка переписок работает.
- **Критерий решения:** сравнение с базовой линией EXP-01 по CM2, не по цене лида.
- **Статус:** planned.

### EXP-04 — Custom B2C → Direct / custom landing

- **Гипотеза:** кастом рекламируется отдельно от готовых SKU и имеет собственную экономику; `Custom CM2 per owner-hour` может оказаться ниже, чем у готовых SKU, из-за правок и мокапов.
- **Первичная метрика:** Custom CM2 per owner-hour (см. MASTER_PLAN_V2 §10).
- **Предусловия:** прайс-сетка кастома, правила депозита, лимит бесплатных правок, событие `custom_print_submit` (или ручной учёт).
- **Стоп-условие:** очередь кастома съедает время владельца, нужное на готовые заказы.
- **Статус:** planned.

## Блок B — Cash-flow и оффер

### EXP-05 — Предоплата 0 / 150 / 200 грн

- **Гипотеза:** депозит снижает невыкуп, но одновременно снижает конверсию в заказ; чистый эффект на CM2 неизвестен и не может быть взят из чужих данных.
- **Первичная метрика:** CM2 на созданный заказ по когорте (учитывает и падение конверсии, и снижение RTS).
- **Дизайн:** последовательные когорты (2–3 недели на вариант) либо разделение по продукту; фиксировать: created → paid deposit → shipped → delivered/RTS.
- **Guardrails:** жалобы/негатив в чатах, доля брошенных оформлений.
- **Критерий решения:** вариант с лучшим CM2 на созданный заказ при достаточной выборке; при малой выборке — «нужно больше данных».
- **Статус:** planned. Для кастома депозит — не эксперимент, а правило (защита от бесплатной работы).

### EXP-06 — Наборы / парные / gift-бандлы

- **Гипотеза:** бандлы повышают средний чек, но их влияние на конверсию и маржу неизвестно; бандл — отдельная продуктовая гипотеза, а не «обязательный апсейл».
- **Первичная метрика:** CM2 на заказ с бандлом vs CM2 на одиночный заказ (с учётом скидки и субсидии доставки).
- **Предусловия:** физическое наличие обоих SKU в наборе (не phantom SKU), рассчитанная маржа набора.
- **Статус:** planned.

### EXP-07 — Порог бесплатной доставки

- **Гипотеза:** порог бесплатной доставки выше среднего чека увеличивает средний чек больше, чем стоит субсидия.
- **Первичная метрика:** CM1 на заказ до/после введения порога.
- **Предусловия:** фактический средний чек и фактическая стоимость доставки по данным TwoComms.
- **Статус:** planned.

## Блок C — Контент и TikTok

### EXP-08 — Скрининг креативных углов

- **Гипотеза:** углы Product-first / Concept-first / Process-first / Founder-first / Pair-Gift-first / Custom-first / B2B proof дают разную экономику; какой работает для TwoComms — неизвестно.
- **Первичная метрика:** на этапе скрининга — стоимость квалифицированного целевого действия; решение о победителе — только по CM2 после накопления заказов.
- **Дизайн:** пакетная съёмка (один съёмочный день → несколько роликов), скрининг в одном ad set, без раздувания структуры. Скрининг ≠ полноценный A/B: он отсекает слабые, но не «доказывает» лучшие.
- **Статус:** planned, повторяющийся.

### EXP-09 — TikTok organic

- **Гипотеза:** органический TikTok можно вести уже сейчас с низкой стоимостью (переиспользование Reels-контента) и он даёт данные о резонансе углов до платного запуска.
- **Первичная метрика:** подписки/сохранения/переходы в профиль как ранние сигналы; заказы — как факт, если появятся.
- **Стоп-условие:** нет (низкая цена), но при нулевом резонансе за 6–8 недель — смена углов.
- **Статус:** can start now.

### EXP-10 — TikTok paid / Spark Ads

- **Гипотеза:** платный TikTok имеет смысл только после подтверждённой экономики Meta и наличия органических роликов-кандидатов для Spark Ads.
- **Предусловия:** EXP-01/02 concluded с положительным CM2; TikTok Pixel/Events API (см. blueprint); свободная мощность производства.
- **Статус:** blocked (по зависимостям).

## Блок D — B2B и DTF

### EXP-11 — B2B «мерч под ключ»: аутрич + лид-форма

- **Гипотеза:** прямой аутрич по ICP + сэмпл даёт B2B-заказы с большей маржой на час владельца, чем B2C.
- **Первичная метрика:** CM2 на закрытую B2B-сделку; повторяемость (второй заказ того же клиента).
- **Предусловия:** ICP, minimum order, прайс, sample-политика; подтверждение бухгалтером допустимости модели в рамках ФОП 2.
- **Статус:** planned.

### EXP-12 — DTF как отдельный продукт

- **Гипотеза:** продажа DTF-плёнки/печати как услуги возможна юридически и экономически.
- **Блокер:** юридическая проверка (КВЭД, договоры, товарная модель, ФОП 2) у бухгалтера/налогового специалиста. До этого — не запускать и не рекламировать.
- **Статус:** blocked (legal).
