## Порядок выполнения (зависимости)

1. **Сначала §0** (TECH-060…066) — без них любые события дают ложные цифры.
2. Затем §1 + TECH-070 (статусная модель) — фундамент атрибуции.
3. TECH-079 (тест-чеклист) — прогнать перед первым запуском Meta.
4. §2 остальное (НП-синк, COGS-снапшот, импорт spend) — по мере появления заказов/рекламы.
5. §3–§7 — по приоритетам P0/P1/P2 из SITE_IMPROVEMENT_BACKLOG.

## Анти-задачи (не делать без явного решения)

- Редизайн, смена CMS/фреймворка, новые поддомены, новая CRM.
- «Оптимизации» без данных из событий или фактических отчётов.
- Калькуляторы/фичи сложнее 4 часов работы владельца.
- Избыточный трекинг «на всякий случай»: каждый новый сбор данных должен отвечать на конкретный вопрос из blueprint §1.
- Скрытый fingerprinting, session replay без маскирования PII и consent.
