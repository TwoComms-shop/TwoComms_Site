## Технические нюансы (сэкономят токены)

- Чеклист — большой файл; НЕ читай целиком. Используй `grep -n 'CB-011' ...` чтобы найти строку
  пункта, потом `sed -n 'N,Mp'` для контекста.
- В `twocomms_global_audit.md` есть повреждённые UTF-8 байты в некоторых строках. Инструмент Edit
  на этом файле МОЖЕТ ЛОМАТЬ кодировку (у меня один раз перезаписал не те байты). Для правок
  чеклиста используй python-скрипт через Bash с байтовым replace и `assert data.count(ob)==1`
  (см. коммиты 87a87235 и предыдущий — там образец).
- Отчёты (`audit_report_*.md`) правятся обычным Edit без проблем.
- Перед коммитом всегда: `git diff --cached | grep -iE 'Trs5m4t|password'` — защита от утечки
  секретов в новые файлы. Пароль в отчётах упоминать только как «пароль в deploy_finance.sh».
- Ветка чата своя (v0/pewemo5000-2082-*), пушь её и мержи PR в main через gh.
