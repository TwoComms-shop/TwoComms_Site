# TwoComms Blog Production Checklist & QA

Этот чек-лист использовать перед сдачей каждой статьи.

## 1. Стратегическая проверка

- [ ] Статья закрывает один главный интент, а не всё сразу.
- [ ] Понятно, для кого статья: B2C, B2B, tech, gift, brand, Instagram-магазин.
- [ ] Статья ведёт к конкретному действию.
- [ ] Статья усиливает `/custom-print/`.
- [ ] Статья не каннибализирует другую статью из списка.
- [ ] Есть связанная внешняя Collaborator-тема.

## 2. SEO-проверка

- [ ] Есть SEO title.
- [ ] Есть meta description.
- [ ] Есть slug.
- [ ] Есть один H1.
- [ ] Главный ключ есть в H1.
- [ ] Главный ключ есть в первом экране естественно.
- [ ] Есть H2/H3 структура.
- [ ] Есть вторичные ключи.
- [ ] Нет keyword stuffing.
- [ ] Есть FAQ 6–8 вопросов.
- [ ] Есть внутренние ссылки.
- [ ] Есть рекомендации по alt-текстам.
- [ ] Есть schema-рекомендации.

## 3. AIO / LLM-проверка

- [ ] Есть короткий прямой ответ на главный вопрос.
- [ ] Есть определения простым языком.
- [ ] Есть списки и чек-листы.
- [ ] Есть таблица, если тема сравнительная.
- [ ] Есть блок `як це робить TwoComms`.
- [ ] Есть уникальные данные/наблюдения/кейсы/визуалы.
- [ ] FAQ ответы короткие и цитируемые.
- [ ] Контент не выглядит как пересказ Википедии.
- [ ] Есть конкретика, которую AI может связать с TwoComms.

## 4. CRO / конверсионная проверка

- [ ] Первый экран не перегружен текстом.
- [ ] Есть 2 CTA: готовый файл и только идея.
- [ ] CTA есть в начале.
- [ ] CTA есть в середине.
- [ ] CTA есть в конце.
- [ ] Есть блок снятия страхов.
- [ ] Есть сценарии использования.
- [ ] Есть визуальные примеры.
- [ ] Понятно, что можно заказать от 1 штуки, если это актуально.
- [ ] Понятно, что можно обратиться без готового макета.
- [ ] Понятно, что TwoComms проверяет детали перед печатью.

## 5. Tone of Voice

- [ ] Текст не звучит как типография 2018 года.
- [ ] Нет канцелярита.
- [ ] Нет чрезмерного streetwear-пафоса в B2C-подарочных статьях.
- [ ] B2B-статьи звучат уверенно и предметно.
- [ ] Tech-статьи звучат инженерно, но понятно.
- [ ] Нет агрессивного очернения конкурентов.
- [ ] Нет пустых лозунгов.

## 6. Юридическая и репутационная проверка

- [ ] Нет фразы `печатаем абсолютно всё без цензуры`.
- [ ] Нет недоказанного `№1`, `лучший`, `самый инновационный`.
- [ ] Нет утверждений вроде `принт никогда не потрескается`.
- [ ] Если упоминаются стирки, есть условие `при правильном уходе`.
- [ ] Если упоминается помощь с файлом, нет обещания бесплатной сложной дизайнерской работы без подтверждения.
- [ ] Если говорится про авторские права, формулировка аккуратная.

## 7. Визуальная проверка

- [ ] Есть список нужных визуалов.
- [ ] Есть alt-тексты.
- [ ] Визуалы соответствуют статье.
- [ ] Первый экран имеет фото/видео.
- [ ] Есть хотя бы один доказательный визуал: макро, before/after, схема, пример.
- [ ] Если визуала пока нет, статья содержит пометку `нужно подготовить`.

## 8. Редактор / функционал

- [ ] Указано, какие блоки нужны в редакторе.
- [ ] Если блока нет, добавлено ТЗ на добавление.
- [ ] CTA-кнопки поддерживаются.
- [ ] FAQ поддерживается.
- [ ] Таблицы поддерживаются.
- [ ] Schema можно добавить.
- [ ] Alt можно прописать.
- [ ] Mobile-версия не ломает структуру.

## 9. Внутренняя перелинковка

Минимум 2–5 ссылок:

- [ ] `/custom-print/`.
- [ ] Каталог футболок.
- [ ] Каталог худи.
- [ ] Страница сотрудничества, если B2B.
- [ ] Размерная сетка, если тема про размер.
- [ ] Уход за одеждой, если тема про DTF/стирку.
- [ ] Статья из этого же кластера.

## 10. Collaborator-проверка

- [ ] Для статьи есть внешняя тема.
- [ ] Есть 3 ссылки.
- [ ] Анкоры не повторяют точь-в-точь другие материалы.
- [ ] Одна ссылка ведёт на блог-статью.
- [ ] Одна ссылка ведёт на `/custom-print/`.
- [ ] Одна ссылка ведёт на релевантный каталог/сотрудничество.
- [ ] Внешняя статья не копирует внутреннюю.
- [ ] Внешняя статья может быть информационной, сравнительной, рейтинговой или B2B.

## 11. Финальный вывод исполнителя

В конце каждой сдачи исполнитель должен написать:

```markdown
## Итог по статье

- Главный интент:
- Главный CTA:
- Основной SEO-кластер:
- Какие визуалы нужны:
- Какие блоки редактора нужны:
- Какой Collaborator-материал должен поддержать статью:
- Что нужно проверить вручную перед публикацией:
```
