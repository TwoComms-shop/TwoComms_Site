# TwoComms: архитектура блог-поста и требования к редактору

## Почему блог-пост должен быть пост-лендингом
Для кастомного принта обычная статья работает слабо. Пользователь часто приходит не за глубоким чтением, а с быстрым эмоциональным интентом:

- хочу футболку с мемом;
- хочу худи с логотипом;
- хочу подарок;
- хочу напечатать фото;
- хочу сделать мерч для команды;
- хочу протестировать принт для магазина.

Если первым экраном дать 3000 символов текста про технологию, человек может закрыть страницу. Поэтому формат должен быть гибридным:

- визуальный верх;
- короткое объяснение;
- быстрые CTA;
- сценарии использования;
- доказательства качества;
- SEO/AIO-блоки ниже.

## Универсальная структура страницы

### 1. Hero-блок

**Цель:** за 5 секунд объяснить, что можно сделать.

Состав:

- H1 с ключом;
- короткое описание 2–3 предложения;
- фото/видео;
- две CTA-кнопки;
- mini trust row: `від 1 штуки`, `DTF-друк`, `футболки/худі`, `допомога з макетом`.

**Пример UA:**

> Є картинка, мем, фото або просто ідея? У TwoComms можна зібрати футболку чи худі зі своїм принтом: вибрати річ, зону друку, завантажити макет або описати задум менеджеру.

**CTA:**

- Завантажити макет
- Описати ідею менеджеру

### 2. Блок снятия страхов

**Цель:** закрыть главные барьеры пользователя.

Карточки:

1. **Від 1 штуки** — можна зробити один принт або тестову річ.
2. **Допомога з файлом** — якщо макет не готовий, підкажемо, що виправити.
3. **Не як наклейка** — підбираємо розмір і розміщення під конкретну річ.
4. **Фото, меми, логотипи** — DTF підходить для повнокольорових зображень.
5. **Футболки і худі** — можна створити базову річ, подарунок або мерч.
6. **Подарунок або бренд** — один сценарий для B2C и B2B.

### 3. Сценарии использования

**Цель:** человек должен увидеть свой мотив.

Карточки:

- для себя;
- для подарка;
- для бренда;
- для команды;
- для Instagram-магазина;
- для локального дропа;
- для мероприятия;
- для физического магазина.

### 4. Как заказать

**Цель:** снизить когнитивную нагрузку перед конфигуратором.

Шаги:

1. Обираєш річ.
2. Обираєш колір і розмір.
3. Обираєш зону друку.
4. Завантажуєш макет або описуєш ідею.
5. Менеджер перевіряє деталі й підтверджує замовлення.

### 5. Визуальное доказательство

**Цель:** показать продукт, а не только обещать.

Форматы:

- фото на человеке;
- мокап;
- видео печати;
- видео упаковки;
- фото макро принта;
- comparison slider;
- before/after;
- примеры хорошего и плохого файла;
- один принт на разных размерах;
- скриншоты конфигуратора.

### 6. Полезный SEO/AIO-блок

**Цель:** дать текстовую структуру для Google/AI и ответы для внимательных покупателей.

Элементы:

- H2/H3 с ключами;
- определения;
- таблицы;
- чек-листы;
- FAQ;
- блоки «як вибрати», «який файл», «який розмір», «як доглядати»;
- уникальные данные/наблюдения TwoComms.

Важно: это не должен быть «SEO-подвал». Блоки должны быть видимыми, аккуратными и реально полезными.

### 7. FAQ

**Цель:** закрыть long-tail запросы и дать AI-системам короткие ответы.

Формат:

- 6–8 вопросов;
- ответы 300–700 символов;
- без воды;
- видимый FAQ на странице;
- FAQPage schema, если редактор поддерживает.

### 8. Финальный CTA

В конце статьи повторить действие:

- создать футболку;
- загрузить макет;
- написать менеджеру;
- рассчитать тираж;
- обсудить партнёрство;
- выбрать худи/футболку в каталоге.

## Функционал, который желательно иметь в редакторе
Если каких-то блоков нет — разработчику/редактору стоит добавить.

### Критично желательно

1. Hero-блок с видео/изображением и двумя CTA.
2. Карточки преимуществ.
3. Галерея изображений.
4. FAQ-аккордеон.
5. Возможность задавать alt для изображений.
6. Внутренние CTA-кнопки внутри статьи.
7. Таблицы.
8. HTML-блок для schema JSON-LD.
9. Блок «похожие статьи».
10. Блок «связанные товары/категории».

### Очень полезно

1. Sticky CTA на mobile.
2. Comparison slider `до/после`.
3. Before/after image block.
4. Блок скриншота конфигуратора с подсказками.
5. Видео autoplay muted loop.
6. Квиз/мини-калькулятор размера принта.
7. Блок «плохой файл / хороший файл».
8. Блок «один принт на разных размерах».
9. Встроенная кнопка Telegram.
10. Автоматическая генерация оглавления по H2.

## Что обязательно проверять перед публикацией

- Title не длиннее 55–65 символов, если возможно.
- Meta description 130–160 символов.
- Один H1.
- Логичная структура H2/H3.
- Первый экран содержит главный ключ естественно.
- Есть ссылка на `/custom-print/` в верхней части страницы.
- Есть 2–5 внутренних ссылок.
- Есть alt у изображений.
- Есть FAQ.
- Есть CTA в начале, середине и конце.
- UA/RU версии имеют отдельные URL.
- hreflang настроен корректно.
- Structured data соответствует видимому контенту.
- Нет недоказанных обещаний.

## Почему это работает

Такой формат снижает когнитивную нагрузку для пользователя, но сохраняет семантический вес для Google и AI-систем. Человек быстро видит продукт и действие. Робот получает текст, структуру, ключи, FAQ, внутренние ссылки и понятные сущности бренда.
