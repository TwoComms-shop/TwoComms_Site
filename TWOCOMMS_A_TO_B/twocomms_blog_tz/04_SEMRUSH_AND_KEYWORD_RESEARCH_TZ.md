## Что смотреть в SEMrush

Для каждого запроса фиксировать:

1. Keyword.
2. Language.
3. Country/region.
4. Volume.
5. KD.
6. CPC.
7. Intent.
8. SERP Features.
9. Есть ли AI Overview / генеративный блок.
10. Questions.
11. Related keywords.
12. Keyword variations.
13. Top URLs.
14. Тип страниц в топе: категория, услуга, блог, маркетплейс, типография, конструктор.
15. Есть ли видео/картинки в выдаче.
16. Есть ли локальный интент: Киев, Харьков, Украина.

## Как принимать решение по статье

### Если запрос коммерческий
Пример:

- футболка зі своїм дизайном;
- худі зі своїм принтом;
- футболки з логотипом.

Статья должна иметь сильный CTA в конфигуратор, примеры и быстрый сценарий заказа.

### Если запрос информационный
Пример:

- як підготувати макет;
- який розмір принту;
- DTF чи шовкографія.

Статья должна содержать таблицу, чек-лист, FAQ и внутреннюю ссылку на заказ.

### Если запрос болевой
Пример:

- чому тріскається принт;
- чому DTF жорсткий;
- чому принт як наклейка.

Статья должна сначала объяснить проблему, затем показать, как TwoComms снижает риск через проверку файла, правильный размер, технологию переноса и уход.

### Если запрос B2B
Пример:

- мерч для бренду;
- футболки з логотипом;
- дропшипінг одягу.

Статья должна говорить про:

- тестовый тираж;
- предсказуемость;
- сроки;
- упаковку;
- повторяемость качества;
- партнёрство;
- расчёт партии.

## Конкуренты для анализа

Искать в выдаче по каждому кластеру, а не заранее фиксировать список. Обычно могут попадаться:

- Maikoff;
- Futbolka.ua;
- Best-Print;
- Singleprint;
- DTF-студии;
- локальные типографии;
- маркетплейсы;
- страницы онлайн-конструкторов.

По каждому конкурентному URL записывать:

- title;
- H1;
- структура H2;
- есть ли цены;
- есть ли конструктор;
- есть ли FAQ;
- есть ли фото/видео;
- есть ли доставка по Украине;
- есть ли «от 1 штуки»;
- есть ли B2B-оффер;
- есть ли требования к макету;
- есть ли реальные кейсы.

## Как искать информационный прирост

Нужно найти, чего нет у конкурентов, и добавить это в TwoComms.

Потенциальные зоны Information Gain:

- сравнение одного принта на S/M/L/XL;
- фото/видео DTF после стирок;
- примеры плохих файлов клиентов;
- тест «принт как наклейка vs нормальный принт»;
- сценарии для Instagram-магазинов;
- готовые идеи подарочных принтов;
- explainers простым языком;
- фото конфигуратора;
- реальные кейсы брендов/команд;
- производственный бэкстейдж.

## Как использовать результаты

После SEMrush-аудита нужно обновить:

- главный ключ каждой статьи;
- вторичные ключи;
- title;
- meta description;
- FAQ;
- внешние анкоры для Collaborator;
- порядок публикации.

Если SEMrush покажет, что один кластер имеет низкий спрос, но сильный B2B-интент, статью всё равно можно писать, если она нужна для конверсии и ссылочного графа.
